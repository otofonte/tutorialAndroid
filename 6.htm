A pasta /res

A pasta 'res' é o local indicado para armazenar as imagens do app.

Essa pasta também pode armazenar os arquivos de temas, as traduções, as músicas e efeitos sonoros...
Todos os 'recursos' do app.
res é abreviação de resources (que significa recursos).

Eu gosto de depenar essa pasta para o app ficar mais leve.

A gente pode excluir a pasta 'menu' que não tem nada dentro dela.

A pasta 'layouts' a gente vai excluir depois de começar a escrever código java.

Eu quero explicar sobre as pastas 'drawable...'

O sistema android permite alterar o 'zoom' para que idosos acessem o facebook e leiam tudo com letras grandes.
São basicamente 4 opções de zoom:
normal (mdpi)
high definition (hdpi)
low definition (ldpi)
extra high definition (xdpi)

e tem também outras definições para tablet.

Tudo isso não serve para o nosso joguinho.
Por que ele deve apresentar a mesma jogabilidade para todas as configurações de zoom.
(por que senão fica mais fácil ou mais difícil se a área do jogo exibida for maior ou menor. O jogo fica mais fácil se o user enxergar o obstaculo vindo de longe. Fica difícil se o user ver o obstaculo logo de repente, sem ver ele vindo de longe.)

Mas o padrão sugerido é armazenar as imagens de tamanhos/definições difetentes nas pastas 'drawable' correspondentes.

Eu costumo deixar somente uma pasta drawable
Deleto todas as variações de definição.
Todas as imagens ficam dentro da pasta 'drawable'.
Inclusive o arquivo 'ic_launcher.png' que é o ícone do app.

Eu coloco esse ícone dentro da pasta 'drawable' e deleto as outras variações de drawable.

Se você quiser mudar o ícone do app substitua esse arquivo.
Se for renomear esse arquivo você precisa renomear também dentro do arquivo manifest.
